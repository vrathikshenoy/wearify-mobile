# Wearify Customer Mobile Migration Decisions

## Native stack

- Latest stable Expo SDK and React Native selected by `create-expo-app`.
- TypeScript strict mode and Expo Router.
- Convex React client for all remote state and realtime subscriptions.
- NativeWind plus a small token module for exact Heritage Rose values.
- Reanimated, Gesture Handler, Safe Area Context, Screens, SVG, Expo Image,
  FlashList, Expo Video, SecureStore, Image Picker, FileSystem, Sharing,
  Linking, Network, Haptics, Splash Screen, Font, Device, and Notifications.
- React Hook Form and Zod for forms.

No React Query, Redux, bottom-sheet framework, dependency injection container,
native background-removal model, or offline mutation database is needed.

## Delivery sequence

1. Configure the native project, providers, theme, assets, and generated Convex
   contract snapshot.
2. Implement session hydration and the public welcome/login/register flow.
3. Implement the authenticated shell and five tab roots.
4. Implement look/product details and every profile child screen.
5. Add upload, export/share, external links, connectivity, deep links, EAS,
   security controls, accessibility, and tests.
6. Compare web and native screenshots at matched phone widths and record every
   accepted difference.

## Release identity

- App name: Wearify
- Android package: `com.phygify.wearify`
- iOS bundle identifier: `com.phygify.wearify`
- Custom scheme: `wearify`
- Associated web host: `wearify-app.vercel.app`
- Convex deployment: supplied only through `EXPO_PUBLIC_CONVEX_URL`

Apple Team ID and Android signing SHA-256 fingerprints are generated by the
release credentials. They are inserted into the Vercel association files before
universal/app-link acceptance testing.
